# AI 2D 游戏素材生成工具 - 产品需求文档

## 1. 产品定位

本产品不是通用 AI 绘图工具，而是深度融入 2D 游戏开发工作流的素材生成与资产管理平台。

核心定位：输入自然语言需求 → 生成可直接导入 Unity、Godot 等引擎的标准化 2D 游戏素材 → 统一管理项目素材库 → 一键导出打包。

与通用 AI 绘图工具的差异：

- **输出为游戏素材，而非美术画作**：生成结果符合游戏引擎的规格要求（透明背景、指定尺寸、锚点/中心点一致、命名规范）。
- **风格一致性保证**：项目级别的风格配置文件约束所有素材的视觉风格，避免多个素材拼凑感强的问题。
- **全流程覆盖**：从需求描述到素材入库到导出，形成闭环，而非单次 prompt → image 的演示。
- **面向工程交付**：素材元数据、版本、格式、路径均结构化存储，支持持续迭代。

## 2. 目标用户

- **独立游戏开发者**：个人或小团队开发 2D 游戏，缺乏美术资源，需要在有限时间内产出风格统一的素材。
- **小型游戏团队**：有程序、策划但无专职美术的团队，需要一个低门槛的素材生产能力。
- **学生开发者**：课程项目、毕业设计等场景，需要快速产出视觉资源。
- **Game Jam 参赛者**：48-72 小时极限开发，时间极其宝贵，需要即刻可用的游戏素材。
- **原型开发者**：验证玩法阶段，不希望因为缺少美术资源而拖延迭代节奏。

用户画像关键词：有游戏开发能力、会写 prompt、不会画画、需要工程化的素材方案而非单张 AI 画作。

## 3. 核心痛点

| 痛点 | 说明 |
|------|------|
| 手动画素材成本高 | 聘请画师或自己学习绘制的时间与金钱成本远超独立开发者承受范围。 |
| 网络素材风格不统一 | 从不同来源下载的素材画风、比例、光照方向各不相同，拼凑后画面割裂。 |
| AI 生成图无法直接用于游戏 | 普通 AI 工具输出的是艺术画作格式（带背景、尺寸随机、无透明通道），需大量手动处理才能导入引擎。 |
| 尺寸和规格不可控 | 无法指定像素级精确尺寸（如 64×64 的 Tile），无法保证锚点位置统一，无法控制朝向（上下左右）。 |
| 多素材风格不一致 | 同一项目多次生成的素材在色调、笔触、比例等维度上难以保持一致，玩家会感知到风格不统一。 |
| 缺少统一管理流程 | 素材散落在各处，无分类、无元数据、无版本管理，项目规模增大后维护困难。 |

## 4. 核心使用场景

### 4.1 生成角色素材

用户描述角色特征（例如"像素风中世纪骑士，32×32，四朝向，带行走动画"），系统生成符合规格的角色 PNG 文件，透明背景，可直接作为 Sprite 导入引擎。

### 4.2 生成道具素材

用户描述物品特征（例如"16×16 的药水图标，扁平化风格，红蓝绿三色变体"），系统按项目风格配置统一生成，确保与同一项目内其他素材视觉一致。

### 4.3 生成地图 Tile

用户描述地形需求（例如"草地、沙地、水域三种地形 Tile，64×64，无缝拼接，俯视视角"），系统按照统一 Tile 规范生成，确保边缘无缝。

### 4.4 生成 UI 图标

用户描述 UI 元素（例如"RPG 风格的技能图标，32×32，圆形底框配内部符号"），系统生成统一风格的 UI 图标素材。

### 4.5 管理项目素材库

用户在一个项目中查看、筛选、分类所有已生成素材，按素材类型、生成时间、状态等维度组织。

### 4.6 导出素材包

用户选择项目中部分或全部素材，指定导出格式（PNG 目录结构 / ZIP 包 / 带元数据 JSON），一键导出，可直接放入 Unity Assets 或 Godot 资源目录使用。

## 5. MVP 功能边界

第一版本实现以下功能：

1. **创建项目**：用户可创建一个游戏项目，设定项目名称和基础信息。
2. **配置项目风格**：为项目绑定风格配置文件，定义画风（像素、扁平、手绘等）、视角（俯视、侧视、等距）、色彩倾向等全局约束。
3. **提交素材生成请求**：用户以自然语言描述素材需求，指定素材类型（角色、道具、Tile、图标）和基本规格（宽高、背景要求等）。
4. **创建生成任务**：系统将需求拆解为生成任务，进入任务队列，用户可查看任务状态。
5. **生成单张 PNG 素材**：调用 AI 模型生成符合规格的 PNG 图像（透明背景、指定尺寸）。
6. **保存素材元数据**：将素材文件路径、尺寸、类型、生成参数、关联项目等元数据持久化存储。
7. **查看素材列表**：在项目下展示已生成的素材，支持按类型筛选。
8. **下载素材**：支持单张素材下载，PNG 格式。

## 6. 非 MVP 范围

第一版本**不包含**以下功能：

- 复杂的角色动画系统（多帧动画编辑、动画状态机）。
- Sprite Sheet 自动拼接（多帧合并为一张切图）。
- Unity / Godot 引擎插件（不通过插件方式集成，仅提供文件导出）。
- 多人协作（不支持项目共享、权限管理、同时编辑）。
- 在线素材编辑器（不支持 Web 端对素材进行像素级编辑）。
- 复杂 Agent 自主规划（不自动生成关卡布局、不自动设计数值体系）。
- 商业级素材审核流程（无审批流、无水印、无版权校验机制）。

## 7. 后续扩展方向

MVP 验证核心流程后，按优先级扩展：

- **批量生成**：一次提交多个素材需求的批量生成能力，减少用户重复操作。
- **Sprite Sheet 输出**：将同类型多帧素材自动拼接为标准 Sprite Sheet，直接适配引擎的切割导入。
- **TileSet 支持**：按 TileMap 规范输出完整 TileSet，支持边缘自动匹配规则。
- **风格参考图上传**：支持用户上传参考图或已有素材作为风格锚点，约束后续生成方向。
- **Unity / Godot 导出模板**：按引擎的资源目录结构自动组织导出，减少手动整理工作。
- **对象存储集成**：素材文件存储迁移至 S3 兼容对象存储，支持大规模素材管理。
- **异步任务队列**：引入 Celery / Redis 队列，支持长时间生成任务的后台处理与状态追踪。
- **多模型路由**：根据素材类型、风格需求、成本预算自动选择最优 AI 模型进行生成。
- **成本统计**：按项目、按用户维度的 API 调用量统计与费用报告。
